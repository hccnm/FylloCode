## Context

当前 session 列表位于 chat 页左侧侧栏，容器宽度来自 `frontend/src/pages/chat.vue` 的 `aside.w-65`。单个条目由 `frontend/src/components/chat/SessionItem.vue` 渲染，当前结构把独立状态点、8x8 agent icon 槽位、标题、时间/turn 数和 hover 三点菜单放在同一水平骨架中。加入 agent icon 后，前导锚点从 1 个变成 2 个，标题被挤到视觉中心之外。

本次改动只处理 renderer 视觉层级，不改变 session 数据模型、排序、选中、菜单动作和 IPC 契约。

## Goals / Non-Goals

**Goals:**

- 在窄侧栏中把 session 标题恢复为第一视觉焦点。
- 保留 agent icon 与运行状态，但让它们成为“会话身份辅助信息”，而不是与标题并列竞争。
- 在缺失 icon、长标题和 hover 菜单出现时保持条目布局稳定。
- 让列表整体更像会话导航，而不是带分隔线的表格行。

**Non-Goals:**

- 不修改 session 排序、搜索、创建、选择、重命名、删除或持久化逻辑。
- 不引入新的过滤、分组、批量操作或拖拽交互。
- 不为 session 条目新增文本化 agent 名称要求。
- 不调整主进程、preload、shared types 或 OpenSpec 之外的契约。

## Decisions

### 1. 采用 `Conversation-first` 两层内容结构

`SessionItem.vue` 应改为清晰的两层内容：

- 第一行：会话标题，作为主视觉元素，占据正文主宽度。
- 第二行：时间戳与轮次数，作为弱化的辅助信息。

这样做的原因是用户进入侧栏时首先要识别“这条会话在聊什么”，而不是先识别 agent 或状态。

备选方案：

- 保持现有结构，仅缩小 icon 和间距：不能解决“多个前导锚点并列抢注意力”的根问题。
- 改成 `Agent-first`：会放大新加 icon 的存在感，但与当前“会话导航”目标相反。

### 2. 将状态指示并入前导媒体区

现有独立状态点应从独立列改为附着在前导媒体区中的单一状态提示，例如：

- icon 槽位角标；
- icon 容器 ring；
- 无 icon 时仍附着在统一的前导占位容器上。

关键约束是：条目左侧只保留一个前导视觉锚点。实现方式可在 Apply 阶段按现有 `@nuxt/ui` 与 Tailwind token 选择，但 SHALL 不再保留“独立状态点列 + icon 列”的双前导结构。

### 3. 保留固定前导占位，但降低媒体权重

`session-management` 现有 requirement 要求 icon 缺失时保留稳定前导占位，这个约束继续保留。实现上前导区应比当前更紧凑，避免空占位比标题更显眼。缺失 icon 时可使用中性背景容器或通用 fallback 图形，但不能让条目左侧出现空洞感或整体错位。

备选方案：

- 完全移除占位：会导致有 icon 与无 icon 的条目文本起始线不一致。
- 保持当前 8x8 大尺寸空槽：稳定但视觉负担过重。

### 4. 列表改为弱卡片导航而非强分隔线列表

`ChatSidebar.vue` 中的 session 列表容器应给条目留出轻间距，并减少 `border-b` 带来的表格感。选中态继续高亮，但更适合作为整块背景/边框提示，而不是只靠平铺底色。这样能让带 icon 的条目看起来像“对象导航项”，而不是“数据行”。

### 5. 更多菜单保持既有交互语义

当前 spec 已定义“三点菜单在悬停时出现”，本次不改变这一交互契约。Apply 阶段可以通过更稳定的占位或绝对定位减小视觉抖动，但不把本次 scope 扩展为菜单显隐语义调整。

## Risks / Trade-offs

- `[视觉判断主观性]` → 通过 requirement 明确层级目标（标题优先、单前导锚点、稳定占位），并在 `pnpm dev` 下做人工验收，避免只讨论“好不好看”。
- `[测试容易绑死 DOM 细节]` → 组件测试只断言稳定的结构语义与关键 data-test，不强绑具体 Tailwind class 全量字符串。
- `[侧栏宽度不足]` → 优先在现有 `w-65` 内完成重排；只有在标题仍明显受压时才允许小幅调整侧栏或容器内边距。
